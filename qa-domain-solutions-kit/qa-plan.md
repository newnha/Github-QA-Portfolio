# 📋 QA Plan
> 신규 프로젝트 시작 시, QA가 무엇을 확인하고 어떤 기준으로 테스트를 준비했는지를 정리하는 문서입니다.

<br>

```
QA-Plan/
# 신규 프로젝트 QA 준비를 위한 기본 계획 문서
|
├─ Overview
|  # 과제 배경, 목표, QA 관점 요약
|
├─ Stakeholders
|  # 협업 역할 및 산출물 흐름 정리
|  ├─ Planning
|  |  # 기획 문서 및 정책 관련 자료
|  |
|  ├─ Design
|  |  # 디자인 산출물 참고
|  |
|  ├─ Development
|  |  # 개발 작업 단위 및 빌드 정보
|  |
|  └─ QA
|     # QA 작업 단위 및 테스트 산출물
|
├─ Schedule
|  # 테스트 단계별 일정 및 리소스 계획
|  ├─ Phase
|  |  # 개발 완료 / QA 준비 / N차 테스트
|  |
|  └─ Timeline
|     # 단계별 기간 및 투입 규모
|
├─ Test-Environment
|  # 테스트 수행 환경 정의
|  ├─ Client
|  |  # OS / 단말 범위
|  |
|  └─ Server
|     # 서버 Phase 및 테스트 설정
|
├─ Test-Scope
|  # 이번 테스트에서 다루는 범위 정의
|  ├─ In-Scope
|  |  # 테스트 대상 기능/영역
|  |
|  └─ Out-of-Scope
|     # 테스트 제외 대상 및 사유
|
├─ Reference
|  # 테스트 이해를 위한 참고 자료
|  ├─ Policy
|  |  # 정책 및 기준 문서
|  |
|  ├─ Spec
|  |  # 상세 명세 요약
|  |
|  └─ Test-Data
|     # 테스트 데이터 관련 메모
|
└─ Issue-Management
   # 이슈 관리 기준 및 흐름
   ├─ Bug-Policy
   |  # 이슈 등록 및 분류 기준
   |
   └─ Known-Issues
      # 잔존 이슈 및 후속 관리 대상
```
