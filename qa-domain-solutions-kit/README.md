# 📦 QA Domain Solution Kit
> 어떤 환경에서도 QA가 문제를 구조적으로 분석하고,테스트 범위와 품질 판단을 어떤 기준으로 결정했는지를 문서로 정리한 워크스페이스입니다.

<br>

```
QA-Domain-Solution-Kit/ # 특정 도메인에서 QA가 문제를 바라보고 판단하는 방식
|
├─ Overview
|  # 이 QA 워크스페이스의 목적과 기본 원칙
|
├─ QA-Plan
|  # 일정·범위·환경·리스크를 정리하는 시작 문서
|  ├─ Schedule & Phase
|  |  # 테스트 단계 및 타임라인
|  |
|  ├─ Test Scope (In / Out)
|  |  # 테스트 포함 / 제외 범위
|  |
|  ├─ Test Environment
|  |  # OS, 단말, 서버 Phase
|  |
|  └─ Risk Notes
|     # 초기 리스크 메모
|
├─ Spec-Review
|  # 기획 명세 리뷰 기록
|  ├─ Review Points
|  ├─ Q&A
|  └─ Decisions
|
├─ Test-Design-Guide
|  # 테스트 케이스 설계 기준 (항상 참고)
|
├─ Test-Cases
|  # 실제 TC 작성 공간
|
├─ Test-Result
|  # 테스트 결과 요약
|
├─ Bug-and-Known-Issues
|  # 이슈 관리 및 잔존 이슈
|
├─ Decision-Log 
|  # QA 판단의 근거 기록
|
└─ Change-Impact-Note 
   # 변경사항이 기존 기능에 미치는 영향 분석
```
